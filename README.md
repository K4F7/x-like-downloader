# X Likes 下载器 (X Likes Downloader)

本脚本提供两种下载模式：**标记点**（只下载新点赞）和 **全量下载**（按上限分批下载并支持续传）。

## 标记点下载机制（默认）

标记点用于记录“上次下载到哪里”。扫描时从最新点赞开始，遇到标记点就停止，这样每次只处理新点赞，省时省力。

### 为什么更方便

- **不重复**：自动跳过已经下载过的旧点赞  
- **更快**：只扫到标记点，不需要翻到底  
- **更稳**：标记点固定在点赞列表中，逻辑清晰

### 如何使用标记点

1. 打开 Likes 页面，启动脚本。
2. 第一次使用：  
   - 点击「自动设置（第一条）」把当前第一条点赞作为标记点，或  
   - 点击「手动选择推文」选择任意一条作为标记点。
3. 点击「开始扫描」下载媒体；完成后标记点会自动更新为**本次扫描的第一条点赞**。
4. 下次扫描时，脚本会从最新点赞开始，直到遇到**上次下载时的最新推文（即旧标记点）**或你手动标记的推文才停止，只下载这段区间内的新内容。

## 全量下载（分批 + 续传）

全量下载会从最新点赞一直扫描到底，但**单次会受“单次上限”限制**，避免触发 X 的下载限额。达到上限后会记录续传点，下次继续从上次进度往后下载。

- 建议单次上限：**200 个媒体/次**（可自定义）  
- 续传点会在全量下载时自动更新，标记点不会被影响  
- 如需重头开始，可清除续传点

### 常见情况

- **标记点丢失**（原推文被删除）：请重新设置标记点。  
- **想重新全量下载**：清除续传点后再扫描。
